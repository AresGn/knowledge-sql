# 📝 Résumé de l'exercice — Projet 6 : Knowledge Management

## Contexte

**Projet 6 du cours de Gestion des Projets Informatiques** (Pr. LANHA, p.19)  
= **Knowledge Management (KM)** = Gestion et partage des connaissances.

---

## Ce qui est demandé

Créer une **application web de Knowledge Management** dont le **contenu porte sur l'un de ces sujets** (au choix) :

- 🖥️ **ATO** — Architecture et Technologie des Ordinateurs
- 🗄️ **SQL** — Bases de données et requêtes
- 💻 **C** — Langage de programmation C

> L'appli est une **plateforme d'apprentissage** : un utilisateur doit pouvoir consulter des fiches, apprendre et naviguer dans les connaissances du sujet choisi.

---

## Contraintes techniques

- ✅ **Pas de vrai backend déployé** — l'appli tourne en local ou hébergée en statique
- ✅ **Pas de base de données** serveur (pas de MySQL, pas de PostgreSQL...)
- ✅ **Stockage local** via un fichier `JSON` ou `localStorage`
- ✅ **Technologie totalement libre** — HTML/CSS/JS, PHP, Python, Java, Node.js, peu importe ce que vous maîtrisez

---

## Features suggérées pour l'appli

| Feature | Description |
|---|---|
| 🏠 Page d'accueil | Présentation de l'appli et des chapitres disponibles |
| 📖 Fiches de cours | Affichage des connaissances par thème/chapitre |
| ➕ Ajouter une fiche | Formulaire : titre, catégorie, contenu, auteur |
| 🔍 Rechercher | Filtrer les fiches par mot-clé ou catégorie |
| ✏️ Modifier / 🗑️ Supprimer | Gérer les fiches existantes |
| 💾 Stockage JSON | Données sauvegardées localement (fichier JSON ou localStorage) |
| 🧠 Quiz (bonus) | Petit questionnaire pour tester les connaissances |

---

## Exemple concret

> Si tu choisis **SQL** → ton appli contient des fiches sur les requêtes SELECT, INSERT, JOIN, etc.  
> Un utilisateur ouvre l'appli, consulte les fiches, peut en ajouter de nouvelles, rechercher un concept précis.  
> Tout est sauvegardé dans un fichier `data.json` ou dans le `localStorage` du navigateur.
